# OpenReader Architecture & Conventions

OpenReader is a text-to-speech document reader built with Next.js (App Router) with support for EPUB, PDF, TXT, MD, and DOCX.

## Core Technologies
- **Framework:** Next.js (App Router), React, TypeScript
- **Styling:** Tailwind CSS, Headless UI (structural only)
- **Database:** Drizzle ORM (supports both SQLite `better-sqlite3` and PostgreSQL `pg`)
- **Storage:** S3-compatible Object Storage (`@aws-sdk/client-s3`)
- **Auth:** Better Auth
- **Background Workers:** Python (`audiobook_worker.py`) and Node.js (`compute-worker/`) via NATS JetStream
- **Testing:** Playwright (E2E), Vitest (Unit)

## Development Conventions

### Backend & API Routes
1. **Logging:** 
   - `console.log` is strictly banned in backend code.
   - Always use `serverLogger` from `@/lib/server/logger`.
   - Strictly follow the format: `serverLogger.info({ event: 'namespace.action', ...context }, 'message text')`.
   - Error-level logging must include a structured error payload: `error: errorToLog(error)`. Do not use top-level `detail` or `errorCode`.
2. **Error Responses:** 
   - Never return raw `NextResponse.json({ error }, { status: 500 })` for terminal backend errors.
   - Use the shared `errorResponse(error, { logger, event, msg, ... })` helper from `@/lib/server/errors/next-response`.
3. **Database:** 
   - DB schemas are exported from `src/db/schema.ts` (which branches to SQLite/Postgres internally).
   - Use strict TypeScript typing; avoid `any` or explicit casting where type-guards can be used.

### Frontend & UI
1. **Styling:** 
   - Rely on semantic design-system tokens. 
   - **Banned Classes:** Literal Tailwind palette colors (e.g., `bg-red-500`), alpha modifiers on theme tokens (e.g., `bg-accent/50`), and arbitrary pixel values for rounded corners or durations.
   - `transition-all` is banned. Use scoped transition utilities.
2. **Components:**
   - Use UI primitives from `@/components/ui`. The old `formPrimitives` and `buttonPrimitives` are removed.
   - Restrict Headless UI imports (`@headlessui/react`) to structural primitives (e.g., Menu, Listbox, Dialog). Do not use it for basic inputs or buttons.
3. **Imports:** 
   - Imports to `@openreader/compute-core` should be limited to specific root or subpath patterns defined in ESLint (e.g., `@openreader/compute-core/types`).

### Python Worker Context
- The `audiobook_worker.py` script listens to a local NATS queue (`audiobooks.gemini.clean`) to perform NLP cleaning of text using Gemini API before TTS generation.

## Testing
- **E2E:** Playwright tests are under `tests/` or run via `pnpm test:e2e`.
- **Unit/Compute:** Vitest tests are run via `pnpm test:unit` and `pnpm test:compute`.
